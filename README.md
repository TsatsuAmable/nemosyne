# nemosyne
## What is nemosyne?
At this time, it is the seed of an idea for how to build data rich vr environments for everyday use that quickly and easily model and represent reasonably faithfully, real-world datasets. 

It's still fuzzy in my head, but the basic premise is to: ###1. map certain specific data topologies to VR artefacts or topoloies that have defined behaviours, interactions and transforms.
###2. Every action that is performed on these VR artefacts should have corresponding effects on the abstracted datasets. 
###3. We should also be able to use each artefact as a building blok to create larger artefacts


I imagine these are not novel ideas, but I'd like to find out how useful following these rules can make the use of VR as a practical daily tool
