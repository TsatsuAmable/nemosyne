# Nemosyne: Data-Native Architecture

## Data as First-Class Visual Entity

**Version:** 2.0.0 Architecture Proposal  
**Status:** Technical Architecture Design  
**Paradigm Shift:** From scene authoring to data embodiment

---

## 1. The Core Philosophy

### The Problem with Current VR Data Viz

**Traditional approach:**
```
Data → Hand-coded visualization → Manual updates
CSV         ↑ Developer             ↑ Maintenance burden
            ↓                     ↓
    "I choose colors, positions, geometries"
```

**Data-native approach:**
```
Data ──────→ Automatic visual derivation ──────→ Living visualization
    │                                               ↓
    └── Topology determines artefact          Self-updating
        Properties determine aesthetics         Semantic meaning
        Relationships determine layout        Emergent structure
```

### The Nemosyne Principle

> "The data is the scene. The developer only curates the mapping."

## 2. The Three-Layer Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│  LAYER 3: PRESENTATION (A-Frame/DOM)                             │
│  ─────────────────────────────────                                │
│  • Rendered entities (a-box, a-sphere, etc.)                      │
│  • Shader materials                                              │
│  • Interaction states (hover, select)                            │
│                                                                  │
│  This layer is AUTO-GENERATED by Layer 2                          │
└─────────────────────────────────────────────────────────────────┘
                              ▲
                              │ "Generate"
┌─────────────────────────────────────────────────────────────────┐
│  LAYER 2: MANIFESTATION (Nemosyne Core)                          │
│  ─────────────────────────────────────                            │
│  • TopologyDetector → Maps data structure → Artefact type         │
│  • PropertyMapper   → Maps data values   → Visual properties       │
│  • LayoutEngine     → Maps relationships → Spatial positions    │
│  • Animator         → Maps temporal changes → Motion             │
│                                                                  │
│  This is the "curator" - automatic transformation                 │
└─────────────────────────────────────────────────────────────────┘
                              ▲
                              │ "Transform"
┌─────────────────────────────────────────────────────────────────┐
│  LAYER 1: ONTOLOGY (Data + Schema)                               │
│  ─────────────────────────────────                              │
│  • Raw data (JSON, CSV, Graph, Time-series)                     │
│  • Schema annotations (semantic types, units, relationships)    │
│  • Context (time, user intent, domain)                          │
│                                                                  │
│  This is the "truth" - just data + meaning                        │
└─────────────────────────────────────────────────────────────────┘
```

## 3. The Ontology Layer (Data Model)

### 3.1 The Nemosyne Data Packet

Every data point entering the system must follow this schema:

```typescript
interface NemosyneDataPacket {
  // Identity
  id: string;
  
  // The payload (can be anything)
  value: Scalar | Vector | TimeSeries | GraphNode | Image;
  
  // Semantic typing for automatic visual mapping
  semantics: {
    type: 'temporal' | 'spatial' | 'categorical' | 'relational' | 'quantitative';
    subtype?: 'timestamp' | 'latlong' | 'category' | 'currency' | 'percentage';
    
    // For color mapping
    domain?: string;        // "temperature", "sales", "risk_level"
    scale?: 'continuous' | 'ordinal' | 'nominal';
    
    // For spatial mapping  
    coordinateSpace?: 'geo' | 'embedding' | 'cartesian' | 'polar';
    dimensions?: number;    // 2D, 3D, nD
    
    // For topology detection
    structure?: 'point' | 'vector' | 'matrix' | 'graph' | 'tree' | 'grid';
    
    // For animation
    temporal?: boolean;
    velocity?: Vector3;
    uncertainty?: number;   // For probabilistic rendering
  };
  
  // Relationships (automatically creates connections)
  relations: {
    parent?: string;              // Hierarchical
    children?: string[];
    links?: { to: string; type: string; weight: number }[];
    temporal?: { before: string[]; after: string[] };
    spatial?: { near: string[]; contains: string[] };
  };
  
  // Context (affects rendering)
  context: {
    timestamp?: number;
    importance?: number;          // Size/prominence
    confidence?: number;          // Opacity/clarity
    source?: string;
    domain?: string;              // "finance", "weather", "biology"
  };
  
  // User overrides (optional - curator interventions)
  overrides?: {
    color?: string;               // Force specific color
    geometry?: string;            // Force specific shape
    position?: Vector3;           // Force specific position
    animation?: string;           // Force specific motion
  };
}
```

## 4. The Manifestation Layer (Transformation)

### 4.1 TopologyDetector → Artefact Selection

```typescript
class TopologyDetector {
  detect(data: NemosyneDataPacket[]): ArtefactType {
    // Analyze data structure to determine visualization type
    
    const structures = data.map(d => d.semantics.structure);
    const types = data.map(d => d.semantics.type);
    
    // Graph majority → Graph visualization
    if (structures.filter(s => s === 'graph').length > data.length * 0.5) {
      return 'nemosyne-graph-force';
    }
    
    // Tree majority → Tree visualization
    if (structures.includes('tree')) {
      return 'nemosyne-tree-hierarchical';
    }
    
    // Temporal + ordered → Timeline/Spiral
    if (types.includes('temporal') && hasSequentialRelations(data)) {
      return 'nemosyne-timeline-spiral';
    }
    
    // High-dimensional → Scatter/Embedding
    if (dimensions > 3 || hasEmbeddings(data)) {
      return 'nemosyne-scatter-semantic';
    }
    
    // Spatial coordinates → Geo visualization
    if (types.includes('spatial')) {
      return 'nemosyne-geo-globe';
    }
    
    // Categorical → Grid/Cluster
    if (types.includes('categorical')) {
      return 'nemosyne-grid-categorical';
    }
    
    // Default
    return 'nemosyne-crystal-default';
  }
}
```

### 4.2 PropertyMapper → Visual Properties

```typescript
class PropertyMapper {
  private mappingRegistry: Map<string, PropertyMapping> = new Map();
  
  registerMapping(dataProperty: string, visualProperty: string, transform: Function) {
    this.mappingRegistry.set(`${dataProperty}:${visualProperty}`, { transform });
  }
  
  map(data: NemosyneDataPacket): VisualProperties {
    const props: VisualProperties = {};
    
    // COLOR from categorical value
    if (data.semantics.scale === 'nominal') {
      props.color = hashToColor(data.value);
    }
    // COLOR from continuous value (heatmap)
    else if (data.semantics.scale === 'continuous') {
      props.color = valueToHeatmap(
        data.value, 
        getDomainRange(data.semantics.domain)
      );
    }
    
    // POSITION from spatial coordinates
    if (data.semantics.coordinateSpace === 'geo') {
      props.position = latLongToCartesian(data.value);
    }
    else if (data.semantics.dimensions) {
      props.position = projectTo3D(data.value, data.semantics.dimensions);
    }
    
    // GEOMETRY from structure
    switch (data.semantics.structure) {
      case 'point': props.geometry = 'sphere'; break;
      case 'vector': props.geometry = 'cone'; break;  // Directional
      case 'matrix': props.geometry = 'box'; break;  // Volumetric
      case 'graph': props.geometry = 'tetrahedron'; break;
    }
    
    // SIZE from importance/confidence
    props.scale = {
      x: 0.5 + data.context.importance * 1.5,
      y: 0.5 + data.context.importance * 1.5,
      z: 0.5 + data.context.importance * 1.5
    };
    
    // OPACITY from uncertainty
    props.opacity = data.context.confidence || 1.0;
    
    // Apply user overrides (curator intervention)
    if (data.overrides) {
      Object.assign(props, data.overrides);
    }
    
    return props;
  }
}
```

### 4.3 LayoutEngine → Spatial Arrangement

```typescript
class LayoutEngine {
  calculatePositions(data: NemosyneDataPacket[], topology: ArtefactType): Map<string, Vector3> {
    
    switch (topology) {
      case 'nemosyne-graph-force':
        return this.forceDirectedLayout(data);
        
      case 'nemosyne-tree-hierarchical':
        return this.treeLayout(data);
        
      case 'nemosyne-timeline-spiral':
        return this.spiralTemporalLayout(data);
        
      case 'nemosyne-scatter-semantic':
        return this.embeddingProjection(data);
        
      case 'nemosyne-geo-globe':
        return this.geographicLayout(data);
        
      case 'nemosyne-grid-categorical':
        return this.categoricalGridLayout(data);
    }
  }
  
  // Force-directed for graph structures
  private forceDirectedLayout(data) {
    // Use d3-force-3d or custom
    const simulation = forceSimulation3D()
      .force('link', forceLink().id(d => d.id).distance(2))
      .force('charge', forceManyBody().strength(-50))
      .force('center', forceCenter(0, 0, 0));
    
    return runUntilConverged(simulation, data);
  }
  
  // Hierarchical tree layout
  private treeLayout(data) {
    const root = d3.stratify()(data);
    const tree = d3.tree().size([2 * PI, 10])(root);
    return treeToXYZ(tree);
  }
  
  // Spiral for temporal sequences
  private spiralTemporalLayout(data) {
    // Archimedean spiral with time on Z
    return data.map((d, i) => {
      const t = d.context.timestamp;
      const angle = (i / data.length) * PI * 4;  // 2 rotations
      const radius = 2 + i * 0.1;
      return {
        x: radius * cos(angle),
        y: radius * sin(angle),
        z: t / 1000000  // Scale time to space
      };
    });
  }
  
  // UMAP/t-SNE projection for high-D
  private embeddingProjection(data) {
    // Run UMAP or use pre-computed embeddings
    return applyUMAP(data.map(d => d.value.embedding || d.value));
  }
}
```

### 4.4 Animator → Temporal Motion

```typescript
class Animator {
  private animationRegistry: Map<string, AnimationPattern> = new Map();
  
  registerPattern(name: string, pattern: AnimationPattern) {
    this.animationRegistry.set(name, pattern);
  }
  
  determineAnimation(data: NemosyneDataPacket): AnimationConfig {
    // Data with velocity → Movement
    if (data.semantics.velocity) {
      return {
        property: 'position',
        type: 'physics',
        velocity: data.semantics.velocity,
        drag: 0.9
      };
    }
    
    // Temporal data → Growth/pulse
    if (data.semantics.temporal) {
      return {
        property: 'scale',
        type: 'pulse',
        period: 2000,
        amplitude: 0.2
      };
    }
    
    // Uncertain data → Flicker
    if (data.context.uncertainty && data.context.uncertainty > 0.5) {
      return {
        property: 'opacity',
        type: 'noise',
        frequency: 0.5,
        magnitude: data.context.uncertainty
      };
    }
    
    // Important data → Steady glow
    if (data.context.importance > 0.8) {
      return {
        property: 'emissiveIntensity',
        type: 'sine',
        period: 3000,
        min: 0.5,
        max: 1.5
      };
    }
    
    // Default: subtle breathing
    return {
      property: 'scale',
      type: 'sine',
      period: 5000,
      amplitude: 0.05
    };
  }
  
  applyAnimation(entity: Entity, config: AnimationConfig, data: NemosyneDataPacket) {
    const aframeAnimation = this.convertToAFrame(config);
    entity.setAttribute('animation', aframeAnimation);
    
    // Store metadata for updates
    entity.dataset.animationSource = JSON.stringify({
      timestamp: data.context.timestamp,
      velocity: data.semantics.velocity
    });
  }
}
```

## 5. The Presentation Layer (Auto-Generated)

### 5.1 The Artefact Factory

```typescript
class ArtefactFactory {
  private componentRegistry: Map<ArtefactType, ComponentDefinition> = new Map();
  
  create(data: NemosyneDataPacket, topology: ArtefactType): Entity {
    // Get component definition
    const componentDef = this.componentRegistry.get(topology);
    
    // Create base element
    const entity = document.createElement(componentDef.tagName || 'a-entity');
    
    // Generate A-Frame attributes from data
    const attributes = this.generateAttributes(data, componentDef);
    
    // Apply to element
    for (const [attr, value] of Object.entries(attributes)) {
      entity.setAttribute(attr, value);
    }
    
    // Add metadata for reverse-mapping
    entity.datum = data;  // Store original data
    entity.topology = topology;
    
    return entity;
  }
  
  private generateAttributes(data: NemosyneDataPacket, componentDef): object {
    const mapper = new PropertyMapper();
    const layout = new LayoutEngine();
    const animator = new Animator();
    
    const visualProps = mapper.map(data);
    const position = layout.calculatePosition(data);
    const animation = animator.determineAnimation(data);
    
    return {
      // Core transform
      position: formatVector(position),
      rotation: formatRotation(data.semantics.velocity),
      scale: formatVector(visualProps.scale),
      
      // Appearance
      geometry: `primitive: ${visualProps.geometry}`,
      material: formatMaterial(visualProps),
      
      // Animation
      animation: formatAnimation(animation),
      
      // Interaction
      class: 'clickable',
      
      // Nemosyne-specific
      'nemosyne-data': JSON.stringify(data.id),
      'nemosyne-topology': data.semantics.structure
    };
  }
}
```

### 5.2 Example Auto-Generated Components

**For Graph Data → nemosyne-graph-force**
```javascript
AFRAME.registerComponent('nemosyne-graph-force', {
  schema: {
    data: { type: 'array' },  // NemosyneDataPacket[]
    iterations: { default: 100 }
  },
  
  init() {
    // Auto-run layout
    this.layout = new LayoutEngine();
    this.positions = this.layout.forceDirectedLayout(this.data.data);
    
    // Create children
    this.data.data.forEach(datum => {
      const node = this.createNode(datum);
      this.el.appendChild(node);
    });
    
    // Create edges from relations
    this.data.data.forEach(datum => {
      if (datum.relations.links) {
        datum.relations.links.forEach(link => {
          const edge = this.createEdge(datum.id, link.to, link.weight);
          this.el.appendChild(edge);
        });
      }
    });
  },
  
  createNode(datum) {
    const entity = document.createElement('a-sphere');
    const visual = new PropertyMapper().map(datum);
    
    entity.setAttribute('position', this.positions[datum.id]);
    entity.setAttribute('radius', visual.scale.x * 0.1);
    entity.setAttribute('color', visual.color);
    entity.setAttribute('class', 'clickable');
    
    // Animation from importance
    if (datum.context.importance > 0.8) {
      entity.setAttribute('animation', {
        property: 'scale',
        to: `${visual.scale.x * 1.2} ${visual.scale.y * 1.2} ${visual.scale.z * 1.2}`,
        dir: 'alternate',
        dur: 2000,
        loop: true,
        easing: 'easeInOutSine'
      });
    }
    
    return entity;
  },
  
  createEdge(fromId, toId, weight) {
    const line = document.createElement('a-entity');
    
    // Visual weight from data
    line.setAttribute('line', {
      start: this.positions[fromId],
      end: this.positions[toId],
      color: weight > 0.7 ? '#00d4aa' : '#444444',
      opacity: weight
    });
    
    return line;
  }
});
```

**For Temporal Data → nemosyne-timeline-spiral**
```javascript
AFRAME.registerComponent('nemosyne-timeline-spiral', {
  schema: {
    data: { type: 'array' }
  },
  
  init() {
    // Automatic spiral layout from timestamps
    const layout = new LayoutEngine();
    this.positions = layout.spiralTemporalLayout(this.data.data);
    
    // Create crystal nodes
    this.data.data.forEach((datum, i) => {
      const crystal = document.createElement('a-entity');
      
      const visual = new PropertyMapper().map(datum);
      const pos = this.positions[i];
      
      crystal.setAttribute('position', pos);
      crystal.setAttribute('nemosyne-memory-crystal', visual);
      
      // Temporal animation - appear sequentially
      crystal.setAttribute('visible', false);
      setTimeout(() => {
        crystal.setAttribute('visible', true);
        crystal.setAttribute('animation', {
          property: 'scale',
          from: '0 0 0',
          to: `${visual.scale.x} ${visual.scale.y} ${visual.scale.z}`,
          dur: 500,
          easing: 'easeOutElastic'
        });
      }, i * 100);  // Staggered appearance
      
      this.el.appendChild(crystal);
    });
    
    // Connect temporal sequence
    new Animator().applyAnimation(this.el, {
      property: 'line-connections',
      type: 'draw-path',
      path: this.positions
    }, null);
  }
});
```

### 6.1 Declarative Mapping Specification

```javascript
// Example: "Show me temperature data as color-coded crystals on a spiral"
const spec = {
  // Input data (just reference)
  source: {
    type: 'websocket',
    url: 'ws://localhost:8080/temperature'
  },
  
  // Topology override (optional)
  topology: 'timeline-spiral',  // Skip auto-detection
  
  // Property mappings (curator choices)
  mappings: {
    // "value" is the domain for color
    'value:temperature': {
      to: 'material.color',
      transform: valueToHeatmap('blue', 'red', {min: -10, max: 40})
    },
    
    // "timestamp" drives position along spiral
    'context.timestamp': {
      to: 'layout.position.z',
      transform: normalizeTimeRange()
    },
    
    // "importance" affects size
    'context.importance': {
      to: 'transform.scale',
      transform: scale => 0.5 + scale * 2
    }
  },
  
  // Layout constraints
  layout: {
    type: 'spiral',
    params: { turns: 3, radius: 5 }
  },
  
  // Animation trigger
  animation: {
    on: 'data-update',
    pattern: 'pulse-scale',
    duration: 300
  },
  
  // Interactions
  interactions: {
    hover: { effect: 'glow', property: 'emissiveIntensity' },
    select: { action: 'show-details', panel: 'info-card' }
  }
};

// Create visualization
const artefact = await Nemosyne.create(spec);
```

## 10. The Philosophy in Practice

**Traditional:**
```
"I need to build a 3D visualization of sales data"
→ Learn Three.js/A-Frame
→ Write 500 lines of positioning code
→ Manually adjust colors
→ Write update logic
→ Maintenance headache
```

**Data-Native:**
```
"I need to see my sales data"
→ Nemosyne.create(data)
→ Done. It looks right because the data is the scene.
```

The developer only intervenes for **curation** (taste, emphasis, narrative), not **construction**.
